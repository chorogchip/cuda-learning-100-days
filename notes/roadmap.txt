GPU Systems & Performance Engineer Roadmap
==========================================

Stage 1 — GPU Fundamentals (Memory / Execution Model)
-----------------------------------------------------

Projects
- vector add
- matrix transpose
- histogram
- reduction
- prefix scan

Concepts to learn
- coalesced global memory access
- shared memory usage
- shared memory bank conflict
- warp divergence
- occupancy
- memory hierarchy (global / shared / register / L2)

Tools
- Nsight Compute
- roofline model
- bandwidth analysis


Stage 2 — Parallel Algorithm Kernels
------------------------------------

Projects
- optimized parallel reduction
- prefix scan (Blelloch scan)
- radix sort
- optimized histogram

Concepts to learn
- warp-level primitives (__shfl_sync)
- warp-level reduction
- block-level reduction
- shared memory layout design
- synchronization (__syncthreads)
- load balancing


Stage 3 — Memory-bound Kernels (Stencil)
----------------------------------------

Projects
- 1D stencil
- 2D stencil (convolution)

Concepts to learn
- halo regions
- shared memory tiling
- memory reuse
- cache behavior
- arithmetic intensity
- roofline interpretation


Stage 4 — GEMM Optimization (Core GPU Kernel)
---------------------------------------------

Projects
- naive GEMM
- shared memory tiled GEMM
- register blocking GEMM
- double buffering
- WMMA / Tensor Core GEMM

Analysis
- compare with cuBLAS

Concepts to learn
- arithmetic intensity
- tile size tuning
- memory pipeline
- tensor core programming
- compute-bound optimization


Stage 5 — ML Kernel Optimization
--------------------------------

Projects
- fused layernorm
- RMSNorm
- softmax
    - safe softmax
    - online softmax
- GeLU
- SiLU

- RoPE
- flash attention

Concepts to learn

- memory bandwidth bottlenecks

- tensor layout effects
- numerical stability


Stage 6 — Triton + PyTorch Integration
--------------------------------------

Projects
- Triton softmax kernel

---------------------------------------------

Projects

Tools
- Nsight Systems
- Nsight Compute

- GPU timeline analysis


Profiling Discipline (Apply to All Projects)
--------------------------------------------

Each project should follow this workflow:

problem
baseline implementation
profiling
bottleneck identification
optimization strategy
implementation
result measurement
hardware-aware explanation

Metrics to report
- latency
- throughput
- memory bandwidth
- occupancy
- speedup vs baseline
